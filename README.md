# 🌿 Smart Plant Pot — IoT + ML Automatic Watering System

An intelligent automatic plant watering system using ESP32, a capacitive soil moisture sensor, and a Random Forest machine learning model. The system reads soil moisture every 60 seconds, sends it to a local Flask server for ML inference, and activates a water pump via relay if the plant needs watering.

## ⚙️ Hardware Required

| Component | Purpose |
|---|---|
| ESP32 DevKit V1 (38-pin) | Main controller + WiFi |
| Capacitive Soil Moisture Sensor | Reads soil moisture via GPIO34 |
| 2-Channel Relay Module | Controls pump via GPIO25 |
| Mini Submersible Water Pump | Delivers water to plant |
| Plastic Pipe (1 meter) | Water delivery tube |
| DC Buck Converter | Steps down 6V → 5V for relay/pump |
| 4x AA Battery Box (6V) | Powers relay and pump |
| 5V USB Power Adapter | Powers ESP32 via Micro-USB |
| Solderless Breadboard + Jumper Wires | Wiring platform |
| Water Reservoir (cup/container) | Holds water for pump |

### Wiring Summary

```
ESP32 GPIO34  →  Soil Moisture Sensor (Signal)
ESP32 GPIO25  →  Relay IN1
ESP32 GND     →  Relay GND (shared ground)
Relay COM+NO  →  Water Pump circuit
Battery Box   →  DC Buck Converter → Relay VCC + Pump
USB Adapter   →  ESP32 Micro-USB
```

## 🚀 How to Run

### Step 1 — Clean the Dataset
```bash
cd server
python eda.py
```
Output: `tarp_clean.csv`, `eda_report.png`

### Step 2 — Train the Model
```bash
python train.py
```
Output: `plant_model.pkl`

### Step 3 — Start the Flask Server
```bash
python server.py
```
Server runs at `http://0.0.0.0:5000`

Check server is running:
```
http://localhost:5000/status
```

### Step 4 — Open the Dashboard
```bash
streamlit run dashboard.py
```
Open browser at `http://localhost:8501`

### Step 5 — Upload Firmware to ESP32

1. Open `smart_pot_esp32.ino` in Arduino IDE
2. Edit WiFi credentials and server IP:
```cpp
const char* WIFI_SSID     = "YOUR_WIFI_NAME";
const char* WIFI_PASSWORD = "YOUR_WIFI_PASSWORD";
const char* SERVER_URL    = "http://YOUR_PC_IP:5000/predict";
```
3. Find your PC IP:
```bash
ipconfig | findstr "IPv4"   # Windows
```
4. Select **Board**: ESP32 Dev Module
5. Select **Port**: COMx (check Device Manager)
6. Hold **BOOT** button → click **Upload** → release when "Connecting..."
7. Open **Serial Monitor** at **115200 baud**

## 🔄 System Flow

```
[Soil Moisture Sensor]
        ↓ ADC (GPIO34)
    [ESP32]
        ↓ HTTP POST (WiFi)
  [Flask Server]
        ↓ ML inference
  [Random Forest Model]
        ↓ JSON response
    [ESP32]
        ↓ GPIO25
  [Relay Module]
        ↓
  [Water Pump] → 💧 Plant
```

---

## 🤖 Offline Fallback Mode

If WiFi or server is unavailable, ESP32 automatically uses threshold logic:

```
moisture > 2000  →  Water the plant
moisture ≤ 2000  →  Do nothing
```

## 📊 API Endpoints

| Endpoint | Method | Description |
|---|---|---|
| `/predict` | POST | Receive sensor data, return watering decision |
| `/status` | GET | Check if server is running |
| `/logs` | GET | Return all sensor log entries |

### Example POST request:
```json
{
  "moisture": 2200,
  "temperature": 30.5,
  "humidity": 75.0
}
```

### Example response:
```json
{
  "water": true,
  "confidence": 0.95,
  "message": "Water now!",
  "timestamp": "16:30:00"
}
```

## 📌 Notes

- ESP32 supports **2.4 GHz WiFi only** — ensure your network is on 2.4 GHz
- Server must be running **before** ESP32 sends data
- `plant_model.pkl` only needs to be generated once unless retraining
- Dashboard reads from `sensor_log.json` saved by the server
